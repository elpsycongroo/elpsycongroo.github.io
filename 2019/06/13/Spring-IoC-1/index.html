Spring IoC容器介绍及实例化流程分析

发表于 2019-06-13 | 分类于 JAVA
本文字数： 17k | 阅读时长 ≈ 15 分钟

本文主要介绍Spring IoC容器-ApplicationContext（BeanFactory）以及它的实例化流程（仅指实例化容器，不包括Bean），通过源码分析IoC容器的创建步骤，加深对Spring框架结构和所用设计思想的理解。

BeanFactory结构介绍

ApplicationContext是最常见的IoC容器接口，三个著名的实现类就是ClassPathXmlApplicationContext、FileSystemXmlApplicationContext和AnnotationConfigApplicationContext， 分别对应XML配置、文件配置和注解配置读取。

ApplicationContext接口继承了ListableBeanFactory接口和HierarchicalBeanFactory接口，第一个接口可以获取多个Bean，第二个接口可以配置BeanFactory的父子关系。

而AutowireCapableBeanFactory可以自动装配Bean，这三个接口直接继承BeanFactory，ApplicationContext通过组合方式使用这个接口。

ConfigurableListableBeanFactory接口实现了上述直接继承BeanFactory的所有三个接口，而ApplicationContext没有。

点进【2.1】处的createBeanFactory()方法：

这里返回了一个DefaultListableBeanFactory实例。从文首的继承关系得知，这个哥们是最完整的BeanFactory实现，因为它通过实现ConfigurableListableBeanFactory接口实现了BeanFactory的所有三个接口，并且通过实现AbstractAutowireCapableBeanFactory接口实现了右侧所有功能。

那么我们可以知道，ApplicationContext接口虽然没有直接继承BeanFactory的所有子接口，但是它内部组合了一个最牛的DefaultListableBeanFactory实例。所有和BeanFactory相关的操作，其实都是委托给这个哥们处理的。

内部是怎么组合的呢？其实ApplicationContext接口定义了一个 getAutowireCapableBeanFactory()方法，该方法返回一个AutowireCapableBeanFactory实例，向下转型就可以得到DefaultListableBeanFactory了。

总结

IoC容器创建的步骤分定位-加载-注册三步.

  1. 定位：资源配置
  2. 加载：解析配置文件，把Bean包装到BeanDefinition里。
  3. 注册：把BeanDefinition对象放入IOC容器中。

还有很多问题没有解决，比如我们之前提到的单例BeanDefinition还没有初始化的问题，以及初始化后“手动”加载的Bean到底是哪些，applicationContext.getBean()方法的运行机制，循环引用在哪里判断等等，这些不属于本文范畴，有空再总结看看。
