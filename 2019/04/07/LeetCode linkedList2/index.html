LeetCode 141 环形链表

给定一个链表，判断链表中是否有环。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

示例 1：

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。

示例 2：

输入：head = [1,2], pos = 0
输出：true
解释：链表中有一个环，其尾部连接到第一个节点。

示例 3：

输入：head = [1], pos = -1
输出：false
解释：链表中没有环。

进阶：

你能用 O(1)（即，常量）内存解决此问题吗？

Solution

经典中的经典，解题思路是快满指针，即一个指针每次指向下一个节点的下一个节点，另一个指针每次指向下一个节点。这样只要链表中有环，它们一定会相遇，并且第二次相遇时慢指针经过的步数就是环的长度。

利用快慢节点同样可以确定一个无环链表的中点，这个后续再提。

LeetCode 142 环形链表II

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

说明：不允许修改给定的链表。

Solution

这两道题可以说是链表环非常经典的题目了。

这道题和上一道题的区别是，需要判断环的入口，假定环的入口距离链表起点为a，那么令第一次相遇时距离链表入口为b，由于快指针比慢指针多走了a+b步，因此慢指针再走a+b步就一定可以到达这次相遇点，而相遇点距离起始点的距离又是b，所以慢指针走a步，就一定可以到链表环的起始点。

注意到起点到环入口的距离也是a，此时安排一个慢指针从起点出发，与原来的慢指针同步前行，它们相遇的点就一定是链表环的起点。

分析结束，代码就很好写了，按照上述思路求指针相遇的节点即可。

LeetCode 143 重排链表

给定一个单链表 L：L0→L1→…→Ln-1→Ln ，
将其重新排列后变为： L0→Ln→L1→Ln-1→L2→Ln-2→…

你不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

Solution

可以看出，该题重点是找到链表的中点，之后将中点后的链表反转后插入到原来的链表中，注意处理中点前后节点的关系。

寻找中间点的方式就是快慢指针法，当快指针到链表尾部时，慢指针指向的就是链表中点。

LeetCode 328 奇偶链表

给定一个单链表，把所有的奇数节点和偶数节点分别排在一起。请注意，这里的奇数节点和偶数节点指的是节点编号的奇偶性，而不是节点的值的奇偶性。

请尝试使用原地算法完成。你的算法的空间复杂度应为 O(1)，时间复杂度应为 O(nodes)，nodes 为节点总数。

Solution

依旧是考察链表的基本操作，本题可以用一个简单点的方式，记录奇偶链表头，然后拼在一起；但是既然题目说了是原地交换，那么就不这么做好了（这么做也是可以过的）。

利用两个指针odd even分别指向头和头的下一个节点，即为奇偶节点，再用一个节点缓存第一个偶数节点；之后将奇数指针的next指向偶数指针的下一个节点，将奇数指针指向该节点，自然就形成了缓存，此时奇数指针的next指向偶数指针指向节点的next指针应该指向的节点，将偶数指针next指向奇数指针指向节点的下一个节点，修改偶数指针指向该节点，反复操作，直到奇数或偶数节点一方为空为止，此时注意将最后一个奇数指针节点的next指向之前缓存的第一个偶数节点，完成链接。
