链表的题目，总结起来，无非就是要保存好上一个节点和当前节点的指针，碰到复杂的链表问题，要利用好白板画图。在进行链表操作的时候，一定要想好上一个节点和下一个节点的关系，切忌想当然而出现断链或者成环之类的操作。其实大部分的链表题目都比较基础，只是考察对于链表的基本操作以及边界条件的处理，能否写出健壮的代码很重要，边界处理不明确时，一定要问清特殊情况的处理。

LeetCode 92 反转链表II

反转从位置 m 到 n 的链表。请使用一趟扫描完成反转。

说明:
1 ≤ m ≤ n ≤ 链表长度。

示例:

Solution

本题是一道非常常规的反转链表基本操作，不过加上了一点点关于起始点和结束点的变化。对于这种题目，熟悉反转链表基本操作，稍微注意一下边界条件，保存起始点和结束点的指针即可（话说这不就是一道改变了起始点和结束点的链表吗？）。

容易想到，思路就是保存开始点的前一个节点，结束点的后一个节点，内部翻转子链表后，和保存的节点相连接即可。

LeetCode 109 有序链表转换二叉搜索树

给定一个单链表，其中的元素按升序排序，将其转换为高度平衡的二叉搜索树。

本题中，一个高度平衡二叉树是指一个二叉树每个节点 的左右两个子树的高度差的绝对值不超过 1。

示例:

Solution

本题最大的慈悲在于链表是有序的并且是升序排列，这样就极大的降低了难度，不需要手写BST了（还挺麻烦的）。由于BST的特性就是中序遍历是有序的，因此本题的顺序就是BST的中序遍历。转换方式就是使用二分法，递归的根据节点的值与它的父节点的大小关系决定放在左子树还是右子树。注意递归的边界判断为右指针是否在左指针的右侧。

LeetCode 138 复制带随机指针的链表

给定一个链表，每个节点包含一个额外增加的随机指针，该指针可以指向链表中的任何节点或空节点。

要求返回这个链表的深拷贝。

示例：

Solution

该题解法分两种，一种是利用Map存储原链表的节点，key是原节点,value是复制出来的空节点。方法是根据链表的next指针遍历节点，将next指针拷贝到新节点（value）上，再回过头遍历随机指针，将map.get(originNode.random)得到的节点赋值给新节点的Next指针。由于通过get方法获得的节点value是新的节点，故也可看做是一种深拷贝。不过该方法的空间复杂度较高。

另一种解法是将新节点插入在原节点的next，再将新节点的next指向原节点的next，这样就相当于将两个链表融合在了一起，然后通过两次从头到尾的复制得到一个融合链表，再分离即可。(传说中的剑指offer原题？)
